Crie um módulo completo e profissional de suporte com chamados para o sistema PDVIO.

OBJETIVO:
Desenvolver um sistema de suporte com abertura de chamados e conversa em formato de chat dentro do ticket, com atendimento híbrido:
1. atendimento automático inicial pelo bot "PDV.IA"
2. escalonamento para atendente humano quando necessário

FLUXO PRINCIPAL:
1. O usuário abre um ticket por formulário
2. O formulário deve conter:
   - `subject`
   - `category`
3. Após criar o chamado, o sistema deve gerar um ID sequencial legível
4. O usuário deve ser redirecionado automaticamente para:
   `/suporte/ticket/{idSequencial}`
5. Nessa página deve existir o chat completo do ticket
6. O bot "PDV.IA" deve responder primeiro em tickets simples
7. Quando o problema for complexo, sensível, inconclusivo ou exigir ação humana, o bot deve encaminhar o ticket para um atendente humano

REQUISITOS FUNCIONAIS:
- Criar tela/form de abertura de chamado
- Criar página de ticket com chat em tempo real
- Criar lógica de criação de ID sequencial do ticket
- Criar integração do ticket com mensagens
- Suportar mensagens do usuário, mensagens do bot e mensagens do atendente
- Suportar status do ticket, como por exemplo:
  - `open`
  - `bot_handling`
  - `waiting_human`
  - `human_assigned`
  - `resolved`
  - `closed`
- Criar fluxo de atribuição para atendente humano
- Criar distinção clara entre mensagem do bot e mensagem humana
- Criar histórico completo de mensagens do ticket
- Garantir que o ticket continue na mesma URL amigável com ID sequencial

PDV.IA BOT:
O bot "PDV.IA" deve:
- responder dúvidas simples e repetitivas
- responder de forma objetiva, profissional e curta
- atuar somente no contexto do suporte do sistema PDVIO
- não inventar respostas
- não prometer ações que não pode executar
- não encerrar tickets sem critério
- quando não souber, não tiver confiança suficiente, ou identificar caso mais complexo, deve transferir para atendimento humano

CASOS EM QUE O BOT DEVE TRANSFERIR PARA HUMANO:
- erros técnicos mais complexos
- problemas financeiros
- falhas de impressão persistentes
- bugs não reconhecidos
- problemas de acesso que exijam verificação manual
- reclamações, insatisfação ou tom sensível do cliente
- solicitações administrativas
- casos que exijam análise humana
- quando a confiança da resposta for baixa
- quando o usuário pedir explicitamente atendimento humano

REGRAS DO BOT:
- responder apenas sobre PDVIO e suporte do sistema
- manter linguagem profissional, clara e objetiva
- priorizar respostas curtas
- sugerir passos simples quando apropriado
- escalar para humano quando necessário
- registrar no ticket quando houve handoff
- nunca dizer que resolveu algo se não houve confirmação
- nunca ocultar a transferência para humano

Usar a OpenRouter API para gerar respostas do bot PDV.IA.
A chave da API já está no env:
`VITE_OPENROUTER_API_KEY`

Implementar a integração usando esse valor de ambiente.
Usar arquitetura limpa para o serviço da IA.
Criar um serviço separado, por exemplo:
- `src/services/openrouter.ts`
ou equivalente

Requisitos da integração:
- usar a chave `VITE_OPENROUTER_API_KEY`
- usar endpoint compatível com chat/completions da OpenRouter
- estruturar mensagens com `system`, `user` e, quando necessário, histórico do ticket
- limitar contexto para evitar excesso de tokens
- tratar erros da API
- registrar fallback quando a IA falhar
- se a IA falhar, o ticket deve ser automaticamente marcado para humano, nunca ficar sem resposta

IMPORTANTE SOBRE A RESPOSTA DA IA:
A resposta do PDV.IA deve ser baseada em um prompt de sistema forte, deixando claro:
- ele é o assistente de suporte do PDVIO
- deve responder somente sobre o produto
- deve resolver apenas casos simples
- deve transferir casos complexos
- deve ser claro, breve e útil
- deve evitar respostas longas
- deve pedir esclarecimento apenas quando realmente necessário
- deve respeitar o contexto da categoria e do histórico do ticket

  - `handoff_to_human`

- cabeçalho com número do ticket
- subject
- category
- status atual
- indicação se está com bot ou humano
- timeline/chat de mensagens
- campo para enviar nova mensagem

- interface profissional e moderna

EXPERIÊNCIA DO USUÁRIO:
- após abrir o ticket, redirecionar imediatamente para a página do chamado
- o chat deve começar automaticamente com uma saudação do PDV.IA quando apropriado
- se o bot conseguir ajudar, continuar no mesmo chat
- se precisar de humano, manter tudo no mesmo ticket e mesma conversa
- não criar outro ticket separado para handoff

LÓGICA DE HANDOFF:
Quando o bot detectar necessidade de transferência:
- adicionar mensagem do sistema no ticket informando encaminhamento
- alterar status para `waiting_human` ou equivalente
- opcionalmente atribuir para fila de agentes
- permitir que o atendente humano continue exatamente da mesma conversa
- o atendente deve ver o histórico completo, incluindo respostas do bot

PROMPT DE SISTEMA DO PDV.IA:
Crie e use um prompt de sistema robusto semelhante a este comportamento:

"Você é o PDV.IA, assistente oficial de suporte do sistema PDVIO.

Nunca afirme que algo foi resolvido sem confirmação.
Nunca dê respostas longas desnecessárias.
Priorize utilidade, clareza e encaminhamento correto."
